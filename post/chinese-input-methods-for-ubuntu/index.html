Overview

As a Taiwanese, Chinese input methods are essential for my daily communication with my friends and family. I used to type Chinese/Mandarin with Microsoft Chinese Pinyin input that is quite good for me. Unfortunately, there is no any alternaltive that has the same performance in Ubuntu or other Linux distro. Reluctantly, we will introduce popular input methods working fine on Ubuntu in this blog, and I will provide the way I followed to install and configure the methods. Hopefully, I would add more information and tests on different input methods in the future in this blog.

Popular input methods

In my understanding, input methods working on Ubuntu require a framework plus a langauge. Normally, a framework supports several different langauges, but we will only focus on Chinese in this blog.

iBus

This is actually an input framework called Intelligent Input Bus. This is widely used by Ubuntu users and also a default framework in Ubuntu 20.04. It supports Chinese inputs, such as Chewing, and corresponding keyboard settings. However, many users comment that the framework is not as stable as another framework, fcitx.

fcitx

According to the comments from PTT, fcitx+chewing seems the most popular option for PTT users because of its stability. In addition, you can definitely run different Chinese input methods within the fictx framework, such as boshiamy and google pinying. Of course, there are many other langauges offered for users, like mozc for Janpanese.

hime

If you really like Microsoft Chinese Pinyin input, hime is highly recommended. Although it is similar to the Microsoft input method, it also provides so many user-defined options. Here a video introducing how to install and make configuration for hime is. However, according to the comments from other ubuntu users, hime has been reported to have many bugs; for example, users are unable to type Chinese by hime in telegram. Besides, the developments of the framework seems stopped which could leave issues unsolved.

gcin

Like other input frameworks, gcin supports many langauges but particularly focuses on “tranditional Chinese”. Although it is one of popular input frameworks on Linux, several bugs have been reported that requires to make additional settings for specific applications. Besides, a fun fact about gcin is that hime was developed by a group of people who was in gcin community.

Installation for iBus-chewing

Installation

The installation steps are quite easy but you may need to take extra steps to fix some problems I listed in the issues section. I will only focus on the normal steps to install and configure the input method.

First of all, one can install the framework and the langauge input method via apt.

sudo apt update
sudo apt-get install ibus ibus-chewing

Configuration

After installation, we need to add the langauge at “Region & Langauge” in “settings”. Press the “+” button in the section of “Input Sources”, and choose Chinese (Taiwan). There should be Chinese (chewing) inside the list. If not, select “Other” option to find chewing. After adding the chewing, Chinese (chewing) should be present in the “Input Sources” section. More importantly, we should switch our framework to ibus that can be chosen in the “manage installed langauges”. Noted that we should install “Chinese (traditional)” by pressing the install/remove langauges button. You may want to take a look at this tutorial with step-by-step screenshots.

Issues

Ubuntu or other Linux distro are really free for users that results in many different issues to fix. Here, I list some issues I met before and the solutions I applied to fix the problems.

No chewing options in settings

This could be the easiest one. You just need to log out or reboot your computer everytime when you install applications.

ibus-chewing only works in document manager

This means your installation and settins are partially correct, but you may need to provide additional information for applications to “sense” the input method. That is, we have to set environment variables for the ibus-chewing. According to the wiki page, we can add a .bashrc file under the /home/ with lines below.

export GTK_IM_MODULE=ibus
export XMODIFIERS=@im=ibus
export QT_IM_MODULE=ibus
export GTK_IM_MODULE_FILE=/usr/lib/x86_64-linux-gnu/gtk-3.0/3.0.0/immodules.cache

Next, enter im-config and choose default for configuration.

This method works for me that allows me to type Chinese in memo, firefox and Google Chrome but Chromium. I guess I could fix it if I reinstall Chromium again.

immodules.cache disappears

If it returns nothing after checking the file with the path, /usr/lib/x86_64-linux-gnu/gtk-3.0/3.0.0/immodules.cache, you need to execute

sudo /usr/lib/x86_64-linux-gnu/libgtk-3-0/gtk-query-immodules-3.0

for generating the file again. However, you may want to try

sudo apt install libgtk-3-0 ibus-gtk3 ibus --reinstall

in case the previous command does not work.

ibus-chewing is not shown in anywhere

This could be resulted from the langauge environment. You may want to try this command according to the Question:

sudo locale-gen zh_TW.UTF-8
