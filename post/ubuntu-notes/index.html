Overview

Ubuntu is a Linux distribution which is an open-source operating system. Compared to other operating systems, Linux supported by a group of qualified developers offers stable and secure but flexible system for users. In other words, users can customize the OS with any open-source software and modifications of OS itself. Because it does not track any digital footprint, Linux is hard to be hacked by using backdoor and infected by virus. Although the features I mentioned above are pretty charming, the Ubuntu’s UI in the desktop version, personally, is not very comfortable and intuitive. Fortunately, we can enjoy using the OS if we are familiar with the uses of Linux’s terminal. In this blog, we are going to introduce useful commands in the terminal, ways to install important software, and issues we could meet$_{[1]}$.

Installation

.deb files

Sometimes we need to download .deb files for some applications. To install it, we can simply apply

sudo dpkg -i package_file.deb

For removing the applictaions installed from .deb files, we can use the command that is the same as removing other files.

sudo apt-get remove [PACKAGE]

Issues

Fix broken packages

  • Bluejeans dependence gconf2 is not installed

If you try to run software updater, you could receive warnings when having any broken packages. Reviewing error messages and logs, you may find specific solutions for that. Otherwise, you can run one of the three recommanded commands below

$ sudo apt-get update –fix-missing
$ sudo dpkg –configure -a
$ sudo apt-get install -f

I solved the issue by applying the last one.

Internal error in Canonical Livepatch (not solved yet)

The Livepatch app on the topbar of Ubuntu keeps showing a warning,

Canonical Livepatch has experienced an internal error. Please refer to https://wiki.ubuntu.com/Kernel/Livepatch#CommonIssues for further information.

There are solutions I’ve tried but eventually failed.

  1. Refresh the app$_{[2]}$

Using the command,

$ sudo canonical-livepatch refresh

I got error messages like I show below

checking for patches
nothing to apply
kernel: 5.4.0-48.52-generic
patch state: ✗ the application caused a crash last time it was applied, check system logs with `journalctl -f -t canonical-livepatch`
patch version: 72.1
  1. Disable and enable the app$_{[2]}$ The strategy is similar to the first one. I applied the commands in a row with my own key:
$ sudo canonical-livepatch disable
$ sudo canonical-livepatch enable [TOKEN]

While I got the message,

Successfully enabled device. Using machine-token: [TOKEN]

the warning still existed.

Unable to make connections to Airpods (not solved yet)$_{[3]}$

To connect to Airpods, setting up bluetooth could be necessary when you fail to achieve that in settings. I found some solutions online for solving the issue:

In terminal, we should shift our current location to /etc/bluetooth and find main.conf for futher settings.

$ sudo vi main.conf

With the command, we can access the script with temporarily administrative power. Then, uncomment the variable ControllerMode and set breder or dual to its value.

ControllerMode = bredr

After that, save the changes and restart bluetooth.

$ sudo /etc/init.d/bluetooth restart
